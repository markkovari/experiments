# JobRunner — React + Rust over Connect-RPC

A small end-to-end demo: a **React (Vite + TypeScript + Tailwind)** frontend talks to a
**Rust** backend over [Connect-RPC](https://connectrpc.com), using Anthropic's official
Rust implementation ([`connectrpc`](https://github.com/anthropics/connect-rust)).

A form submits a job; the server runs an N-step job and reports progress back **two ways
over the same protocol**:

| Mode | RPC(s) | Pattern |
|------|--------|---------|
| **Streaming** | `RunJobStreaming` (server-streaming) | server pushes one update per step |
| **Polling** | `StartJob` + `GetJobStatus` (unary pair) | client polls status on a timer until DONE |

A single `proto/jobrunner/v1/jobrunner.proto` is the source of truth for both sides.

## Layout

```
connect-rpc/
├── proto/          # jobrunner.proto + buf config (source of truth)
├── server/         # Rust crate: connectrpc + axum, port 8088
│   ├── build.rs    #   connectrpc-build generates Rust types from the proto
│   └── src/main.rs #   JobRunnerService impl (streaming + polling) + job store
└── web/            # Vite app, port 5173
    ├── src/gen/    #   TypeScript generated by buf (@bufbuild/protoc-gen-es)
    └── src/        #   client.ts, App.tsx, StreamingView, PollingView
```

## Prerequisites

- Rust toolchain (`cargo`)
- `protoc` (`brew install protobuf`) — used by `connectrpc-build` in `build.rs`
- Node.js + npm
- `buf` — only needed to **regenerate** the TS client (`make gen`); the generated
  output is committed, so a plain run doesn't require it

## Run

```sh
make install      # one-time: install web deps
make dev          # starts the Rust server (:8088) + Vite (:5173)
```

Open <http://localhost:5173>, pick **Streaming** or **Polling**, submit the form.

Or run the two halves separately:

```sh
make server       # terminal 1 — Rust server on :8088
make web          # terminal 2 — Vite dev server on :5173
```

In dev, Vite proxies the Connect paths (`/jobrunner.v1.JobRunnerService/*`) to the Rust
server, so the browser makes same-origin requests — no CORS. The server also sends
permissive CORS headers for a standalone production build (set `VITE_API_BASE_URL` to the
server origin when building).

## Regenerating code

- **Rust**: regenerated automatically on `cargo build` (via `server/build.rs`).
- **TypeScript**: `make gen` (runs `buf generate` → `web/src/gen`).

Edit `proto/jobrunner/v1/jobrunner.proto`, then rebuild/regenerate both.

## Quick check without the UI

Unary (Connect JSON):

```sh
curl -XPOST localhost:8088/jobrunner.v1.JobRunnerService/StartJob \
  -H 'content-type: application/json' -d '{"label":"x","steps":5}'
# → {"id":"..."}
```

Server-streaming requires the Connect envelope framing, so it's easiest to observe in the
browser's Network tab (a single long-lived POST to `RunJobStreaming` returning streamed
JSON frames).
